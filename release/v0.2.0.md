## discord-agent-hub v0.2.0

Knowledge-source release for retrieval-grounded agents in Discord.

### Highlights

- Added reusable knowledge sources:
  - `/knowledge-import`
  - `/knowledge-list`
- Agent Markdown now supports `knowledge_sources`
- Added `hub_lexical` retrieval over extracted document text
- Added provider-native knowledge backends:
  - `openai_file_search`
  - `gemini_file_search`
- Knowledge import supports `overwrite:true` for replacing an entire source
- Agent displays now show bound knowledge sources
- Anthropic responses now use a larger `max_tokens` value and prompt caching
- `/session-show` now displays cache-related token usage
- OpenAI and Gemini cached input token usage is normalized when providers expose it

### Notes

- `hub_lexical` works across OpenAI, Anthropic, and Gemini agents.
- `openai_file_search` is OpenAI-specific and stores OpenAI vector store IDs in SQLite.
- `gemini_file_search` is Gemini-specific and stores Gemini File Search store names in SQLite.
- Gemini native File Search requires the `google-genai` dependency; re-run `pip install -e .` after updating.
- Provider-native backends are best used with provider-specific source IDs, such as `gpt-papers-openai` and `gpt-papers-gemini`.

### Platform Status

- Tested:
  - macOS
  - Ubuntu server operation with OpenAI, Anthropic, and Gemini APIs
  - OpenAI native file search with GPT agents
- Untested:
  - Windows
  - multi-process deployment
  - Gemini native File Search in production use
