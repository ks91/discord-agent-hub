## discord-agent-hub v0.1.2

Operational maintenance release for multi-server and server-hosted deployments.

### Highlights

- Global application commands are now synced even when `DEV_GUILD_ID` is set
- Added `scripts/hubctl.sh` for `start`, `stop`, `restart`, `status`, and `logs`
- Documented `DATA_DIR` usage for running multiple hub instances on one host
- Standardized the hub control script on `bash`

### Notes

- For normal operation across multiple Discord servers, leave `DEV_GUILD_ID` unset.
- If you run multiple hub instances on one machine, set a different absolute `DATA_DIR` for each instance.
- Uploaded documents are text-extracted; embedded images inside documents are not extracted yet.

### Platform Status

- Tested:
  - macOS
  - Ubuntu server operation with OpenAI, Anthropic, and Gemini APIs
- Untested:
  - Windows
  - multi-process deployment
