## discord-agent-hub v0.1.0

First usable release of `discord-agent-hub`: an API-first multi-user Discord hub for OpenAI, Claude, and Gemini agents.

### Highlights

- OpenAI Responses API, Anthropic Messages API, and Gemini API support
- File-based agent management:
  - `/agent-import`
  - `/agent-show`
  - `/agent-delete`
- Discord thread-bound multi-user sessions
- Structured event logging to `data/events.jsonl`
- Persistent session and message storage in SQLite
- Provider-side tools:
  - web search
  - code execution
- Image attachment support for OpenAI, Claude, and Gemini
- Document text extraction for:
  - `.txt`
  - `.md`
  - `.csv`
  - `.pdf`
  - `.docx`
  - `.pptx`
  - `.xlsx`
- Hidden-instructions agent option:
  - `public_instructions: false`
- Session inspection and export:
  - `/session-show`
  - `/log-export`
- Lightweight operational usage summary:
  - `/usage-report`
- Runtime safeguards:
  - per-thread serialization
  - provider timeout and retry
  - queue visibility in structured logs
  - role-based access restrictions via `DISALLOWED_ROLE_IDS`

### Notes

- The hub currently assumes a single bot process on one host.
- Uploaded documents are text-extracted; embedded images inside documents are not extracted yet.
- `loglm` importer work remains on the `loglm` side.

### Platform Status

- Tested:
  - macOS
- Untested:
  - Linux server / VM deployment
  - Windows
  - multi-process deployment
